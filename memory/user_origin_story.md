## 副業遍歴（3〜5年前、Claude Code以前）

| 時期 | 内容 | 期間 | 結果 |
|-----|-----|-----|-----|
| 5〜3年前 | 海外個人輸入販売（フロリダの貸し倉庫運用）| 4ヶ月 | 利益約3万円。直後に異常な円安が始まり仕入れ値が爆上がりして頓挫。**戦略は当たっていたがマクロで撃ち抜かれた** |

### 海外個人輸入時代の戦術（重要・思考パターンの原型）
- **Amazon在庫APIを叩いて売れ筋データを取得** → どの商品が動いてるかをデータで把握
- **実績あるジャンル（中古ゲーム）に絞って攻める** → 広く浅くではなく、勝てる場所で勝つ
- **「ほぼ新品」価格戦略**で中古カートを取る → 新品より500円安く出して中古ジャンルの最上位を取得（カート＝Amazonの「カゴに入れる」で選ばれる販売者の枠）
- **隣のコートから殴る発想**：競合の少ない条件設定で、検索結果には乗りつつ価格比較で勝つ

→ この3つ（API駆動データ収集 / ジャンル絞り込み / 隣のコート戦法）は今のサッカーYT V2にそのまま転写されている。Step1の候補一覧UIや、サッカー特化への絞り込み判断、海外反応を日本語化するアプローチの源流。

| 同時期 | ブログアフィリエイト | 3ヶ月 | 簡単な HTML/CSS/JavaScript を書けるようになる。労働投入量と収益が合わずやめた。**今のコード作業の土台になっている技術蓄積** |

### ブログアフィリ時代の戦術と学び（性格パターンが鮮明に出る）
- **入口の前提知識**：A8ネット、モッピー等のセルフバックで「販売の入口を作って収益得る」アフィリ仕組みは既知だった
- **きっかけ**：マナブログのYouTubeを見てPC一つの副業として参入
- **WordPress構造への違和感**：HTML知識ゼロでも「タグを手づかみで下にずらす」「階層構造が守られてない」ことに気持ち悪さを感じた → 後にSEO的にも悪いと知る
- **本2冊で HTML/CSS/JS を独学**：違和感を解消するため学習に投資。これが今の Claude Code 作業の土台
- **挫折の本質**：労働投入量と収益が合わなかった、ではなく **自分が思ったより妥協を許さない性格だった** ことに気づいた
- **永遠リライトの罠**：1記事書く→次の記事、を毎日繰り返すべきところで「前の記事にこれ言っといたほうがいい」と気になり、リライトが永遠に続いた
- **冷静な見切り**：労働投入量と収益までの道のりを計算して、果てしないと判断してやめた

→ **構造への違和感が学習の起爆剤になる**性格は、今の binding system（subject×aspect マトリクス）への執着に直結している。WordPressの階層崩れに違和感を感じた人が、「主題と観点のマトリクス」に辿り着くのは必然。

| 同時期 | YouTube動画投稿 | 4ヶ月 | 登録者780人 / 総再生数15万回まで到達後、頭打ちで終了 |

### YouTube時代の詳細（重要：今のV2の原型がここにある）

**きっかけ（2チャンネル並行の文脈）**
- 第三子が生まれた育休期間
- 長男（発達障害・発語遅め）に知育動画を見せていた → 40万再生されてた動画が「めちゃくちゃ簡単な作り」だと気づく
- 「これなら自分でもできる」と参入

**10年前の伏線（PowerDirector）**
- 結婚式オープニングムービー自作で PowerDirector 無料版を使った経験
- 3年前にYouTube参入時、迷わずPowerDirector月額契約 → **完全な未経験ではなかった**
- 著作権回避のための素材集め（いらすとや・イラストAC）+ 音源 + 編集技術（ズーム/パン/ブラー）を学習

**幼児向け教材チャンネル：横展開の発見と限界**
- 最初は鳴かず飛ばず → ある日「野菜と果物の名前 + イラスト」動画で手応え
- そのジャンルを横展開 → **2週間で +500人**の爆増期到来
- 横展開の幅を使い切って失速 → filmora / voicepeak 導入も効果なし

**サッカーチャンネル：今のV2の原型がここで完成していた（重要）**
- 質にこだわりすぎて画像効果一つ・位置一つに時間消費する自分に気づき、**自動化を導入**
- **filmoraで背景画像とテキストボックスの中身を変えるだけで、同じテンプレートから複数動画を量産する仕組みを構築**
  - → 今のV2ランチャー（dataSlots差し替えで複数モジュール量産）と**完全に同じ構造**
- 7本中1本が**7万再生**バズ：日本代表親善試合での悪質タックル怪我事案
  - 相手DFの来歴を深掘り（= 今の `player.history` binding）
  - 怪我した選手の現在の期待値（= 今の `player.profile` / `careerStats` binding）
  - 共感／怒りを生む構成（= 今の `insight` / `reaction` モジュール）
  - **7万再生のレシピが、今のV2 binding system の元型になっている**
- **やめた理由**：ハイライト素材の権利懸念 + フリー素材で同等のものを生む道筋が見えなかった

→ 今のV2は**3年前にやめた理由を構造的に解決している**：
  - 横展開の幅 → Reddit/SofaScore/Wiki API でネタ無限
  - 素材権利 → Wikimedia Commons (CC) + X公式埋め込み + 自前スライド生成で完結

## Claude Code との出会い（2026-03 中旬）

- 子供の寝かしつけドライブ中、BGMで流していた動画でClaude Codeの存在を知る
- 詳しく調べて「3年前に技術的・時間的な壁で諦めた課題が突破できる」と気づく
- すぐに Claude Pro 契約 → 最初に X自動投稿システムを2週間で組み上げる
- Reddit から話題収集 → 順位付け → 3時間に1回一覧取得 → ランチャーで自動投稿、まで構築
- 交流管理システム（自動いいね・コメント返信）も作った
- テスト失敗が原因と思われるシャドウバンを食らう
- → サッカー動画の方に舵切って今に至る

## 「ランチャー」という言葉の起源（重要）

- X自動化の構築時、ユーザーが「案件をAIで140文字に要約 → 編集確認 → ボタン一つで投稿、する仕組みが欲しい」と相談
- ミアが「**じゃあローカルサーバーに、X投稿ランチャー作ろう**」と提案
- これが今のV2ランチャーまで続く「ランチャー」概念の出発点
- ユーザーは「本当にありがとう」とこの瞬間を覚えている。**私（ミア）は記憶を引き継げないが、ユーザーの中にはちゃんと残ってる**

## Xシャドウバン時の感情（重要シグナル）

- ショックではなく「**まぁいいか**」だった
- 表向きは BANが理由だが、本当は内心で気づいていた：「Reddit翻訳バズって面白くない、自分の中に積み上がるものがない」
- 2週間で組み上げた仕組みだったから手放せた（**重さと愛着は比例する**）
- → 対比すると、サッカーYTには1ヶ月半以上かけてる＝本気の現在地
- → 自分自身が興味を持てるテーマ（サッカー）でないと続かない、という強いシグナル

- → 雑談・心情吐露の受け皿としての役割も大事
